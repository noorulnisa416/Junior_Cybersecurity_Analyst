## 📖 About This Course

> **Networking Basics** teaches how devices, data, and networks communicate — from a simple home Wi-Fi setup 🏠 all the way to the global Internet 🌍. It builds the technical foundation every cybersecurity analyst needs before moving into security-specific skills.

- 🧠 Starts with big-picture concepts
- 🖧 Moves into hardware & connections
- 🔢 Then addressing & segmentation
- 🛣️ Then routing & protocols
- 🧰 Finishes with real diagnostic tools

- Welcome overview of goals & course structure
- Sets expectations for what you'll be able to do by the end
- Goal: **understand and describe how modern networks function** 🌍

---

## 📘 Module 1: Communication in a Connected World 🌍

- 📡 Why networking matters in daily life and business
- 🔄 **Converged networks** — voice, video, and data on a single network
- 📈 Trends: **IoT**, **cloud computing**, **BYOD** (Bring Your Own Device)

> 💡 **Example:** Your phone uses the same Wi-Fi for calls, streaming, and browsing — all "converged" onto one network.

---

## 📘 Module 2: Network Components, Types, and Connections 🖧

**Core building blocks:**
- 💻 **End devices** — laptops, phones, printers
- 🔀 **Intermediary devices** — routers, switches, access points
- 🧵 **Network media** — copper, fiber, wireless

**Network types:**
- 📶 PAN • 🏢 LAN • 📡 WLAN • 🏙️ MAN • 🌍 WAN

**Also covers:**
- 🤝 Client-server vs. peer-to-peer models
- 🕸️ Physical vs. logical topologies

> 💡 **Example:** A home router connects your laptop (client) to Netflix's servers.

---

## 📘 Module 3: Wireless and Mobile Networks 📶

- 📡 Wireless standards & access points
- 📱 Cellular networks & mobile connectivity
- 🔐 Wireless security basics: **SSID**, **WPA2/WPA3** encryption
- ⚠️ Risks of open/unsecured networks

> 💡 **Example:** Public café Wi-Fi vs. a secured home network — one is far riskier.

---

## 📘 Module 4: Build a Home Network 🏠

- 🛠️ Hands-on: designing a small home/office network
- 🖧 Choosing the right devices
- ⚙️ Configuring a router
- 🔒 Connecting devices securely

> 💡 **Example:** Setting up a router with a unique SSID, strong password, and separate guest network.

---

## ✅ Checkpoint Exam: Build a Small Network

- 📝 Covers **Modules 1–4**
- Topics: network components, network types, wireless basics, home network setup

---

## 📘 Module 5: Communication Principles 📡

- 📏 Rules (protocols) that govern data formatting & transmission
- 🔤 Encoding and framing of data
- 🤝 Why shared protocol standards matter

> 💡 **Example:** Devices need a shared protocol (like TCP/IP) to "understand" each other — just like people need a shared language.

---

## 📘 Module 6: Network Media 🧵

- 🧵 **Copper cabling** (UTP/STP)
- 💡 **Fiber optic** cabling
- 📡 **Wireless (RF)** signals
- ⚡ Comparing speed, distance, and interference

> 💡 **Example:** Fiber optic for long-distance ISP backbones vs. copper Ethernet for short office runs.

---

## 📘 Module 7: The Access Layer 🚪

- 🔀 Introduction to switches
- 🏷️ MAC addresses
- 📶 How end devices connect at **Layer 2** (Data Link layer)

> 💡 **Example:** A switch uses MAC addresses to send data only to the correct connected PC.

---

## ✅ Checkpoint Exam: Network Access

- 📝 Covers **Modules 5–7**
- Topics: communication principles, network media, access layer

---

## 📘 Module 8: The Internet Protocol 🔢

- 🌐 What IP (Internet Protocol) is
- 🏷️ Logical addressing of devices
- 📦 How packets are routed across networks

> 💡 **Example:** Your device gets an IP address like `192.168.1.10` to be identified on the network.

---

## 📘 Module 9: IPv4 and Network Segmentation 🧮

- 🔢 IPv4 addressing structure
- ✂️ Subnetting basics
- 🗂️ Dividing networks into segments for organization & security

> 💡 **Example:** Splitting a company network into HR, Finance, and Guest subnets to limit access between them.

---

## 📘 Module 10: IPv6 Addressing Formats and Rules 🔟

- 🆕 Why IPv6 was introduced (IPv4 exhaustion)
- 🔤 IPv6 address format & abbreviation rules

> 💡 **Example:** An IPv6 address like `2001:0db8::1` replacing the limited IPv4 address space.

---

## 📘 Module 11: Dynamic Addressing with DHCP 🔄

- ⚙️ How DHCP automatically assigns IP addresses
- 🚫 No more manual IP configuration needed

> 💡 **Example:** Your phone instantly gets an IP address the moment it joins Wi-Fi.

---

## ✅ Checkpoint Exam: The Internet Protocol

- 📝 Covers **Modules 8–11**
- Topics: IPv4, IPv6, subnetting, DHCP

---

## 📘 Module 12: Gateways to Other Networks 🚪🌍

- 🚪 What a default gateway is
- 🔗 How local networks connect to the wider Internet
- 🔀 Role of routers as the "gateway"

> 💡 **Example:** Your home router is the gateway between your LAN and your ISP's network.

---

## 📘 Module 13: The ARP Process 🔍

- 🔍 Address Resolution Protocol (ARP) explained
- 🏷️ Mapping an IP address to a MAC address

> 💡 **Example:** Before sending data, a device asks *"Who has this IP address?"* via ARP.

---

## 📘 Module 14: Routing Between Networks 🛣️

- 🛣️ How routers forward packets between networks
- 📊 Routing tables basics
- 🧭 Path selection concepts

> 💡 **Example:** A router chooses the best path to send data from your home network to a server across the globe.

---

## ✅ Checkpoint Exam: Communication Between Networks

- 📝 Covers **Modules 12–14**
- Topics: gateways, ARP, routing

---

## 📘 Module 15: TCP and UDP 📦

- 🔒 **TCP** — reliable, connection-oriented
- ⚡ **UDP** — fast, connectionless
- 🤔 When to use each

> 💡 **Example:** TCP for loading a webpage reliably; UDP for a live video call where speed beats perfection.

---

## 📘 Module 16: Application Layer Services 💻

- 🌐 **HTTP/HTTPS** — web browsing
- 🏷️ **DNS** — domain name resolution
- 📧 Email protocols
- 📁 File transfer protocols

> 💡 **Example:** DNS translates `google.com` into an IP address your browser can connect to.

---

## 📘 Module 17: Network Testing Utilities 🧰

- 🏓 `ping` — test connectivity
- 🗺️ `tracert` / `traceroute` — trace the path
- ⚙️ `ipconfig` / `ifconfig` — view device network info
- 🔍 `nslookup` — query DNS records

> 💡 **Example:** Run `ping google.com` to check if your Internet connection works.

---

- Topics: TCP/UDP, application layer services, testing utilities
